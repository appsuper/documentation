Introdução

Olá parceiro desenvolvedor, é um prazer tê-lo conosco!

Este documento explica como se integrar e utilizar a plataforma do App Super para montar o seu e-commerce da forma mais rápida e prática.

Nessa documentação, você encontra todas as informações necessárias para realizar requisições HTTP nos nossos endpoints e gerenciar completamente seus estabelecimentos, seus pedidos, seus produtos e a autenticação com a nossa API.

Nossa plataforma possibilita aos parceiros os seguintes serviços via API:

1. Estabelecimentos

  • Acessar sua lista de estabelecimentos
  • Acessar detalhes de um estabelecimento específico

2. Pedidos

  • Acessar lista de pedidos por estabelecimento
  • Acessar detalhes de um pedido específico
  • Fazer cancelamento de itens e um determinado pedido
  • Cancelar os pagamentos de um pedido
  • Finalizar o pedidos com informações enviados do PDV (Captura de pagamentos)

3. Produtos

  • Acessar detalhes de um produto através do SKU (Código Reduzido do Produto)
  • Acessar lista de produtos de um estabelecimento específico
  • Criar um lote de importação
  • Acessar lista de lotes de importação
  • Acessar detalhes de lotes de importação
  • Acessar detalhes de um produto pelo identificação (ID)
  • Atualizar um produto específico
  • Deletar um produto

Além disso, nossos parceiros têm acesso ao nosso painel exclusivo, onde podem acompanhar pedidos, ver relatórios de performance e muito mais.

As referências aos endpoints da nossa API são mostrados nesse documento no formato abaixo, clique sobre o método http para expandir as informações de consumo do endpoint:

  Método HTTP   Descrição da funcionalidade do endpoint.

/endpoint/path/{path_parameter}/

Informações Iniciais

  1. Separador decimal
    2.

    O separador decimal para todas os campos tipo decimal é o ponto (.).

  2. Produtos do banco de desenvolvimento
    3.

    A lógica por trás da criação de um lote de importação de produtos funciona da seguinte forma. Se um produto enviado na lista existe no nosso bancco de dados, esse produto é criado no seu mercado, caso contrário o produto deve ser catalogado pelo painel de parceiros. Disponibilizamos uma planilha com alguns produtos para teste, você pode acessá-la nesse link.

  3. Porções de produtos à granel
    4.

    Os produtos à granel são vendidos em porções no App Super. Um cliente compra, por exemplo, banana de 1000 em 1000 gramas, e o coletor no mercado separa uma quantidade do produto com peso mais próximo possivel da quantidade comprada pelo cliente. O tamanho da porção é predefinido pela configuração padrão do produto que se encontra no nosso banco de dados, portanto não há um campo que define essas porções no endpoint para criação do lote de importação. O parceiro pode, uma vez importado o produto para o respectivo mercado, alterar o tamanho da porção que o cliente comprará em cada produto por meio do painel administrativo.

Autenticação

A autenticação em nossa API é feita através de uma API key[1], fornecida através do cadastro de parceiro. Essa chave deve ser enviada no header de todas as requisições para que o consumo dos endpoints seja bem sucedido. A Figura 1 mostra de forma representativa o processo de autenticação.

Figura 1: Ilustração da autenticação com a API, a API Key deve ser enviada no header de todos os requests.

Ambiente de Desenvolvimento

O App Super fornece um ambiente de desenvolvimento para testes e homologação do fluxo de trabalho. Você pode acessar esses ambientes por meio dos domínios:

  • Desenvolvimento: http://parceiro.dev.appsuper.com.br/
  • Produção: https://parceiro.appsuper.com.br/

Exemplo:

$ curl -H 'Api-Token: YOUR_API_TOKEN_HERE' -H 'Api-Secret-Key: YOUR_API_SECRET_KEY_HERE' http://plataforma.dev.appsuper.com.br/market/

Atenção

Para acionar os nossos endpoints de forma satisfatória, é necessário adicionar a barra ("/") no final das urls de cada endpoint, por exemplo/market/{market_id}/order/

    Em alguns estabelecimentos existe o conceito de EAN de entrada. Isto é, alguns produtos possuem um EAN principal que se converte em EANs derivados. Isso acontece geralmente com produtos pesados. Importação de Produtos

Para realizar a atualização de preços e características dos produtos no nosso banco, o parceiro deve enviar periodicamente lotes de importação. Os lotes podem ser enviados a qualquer momento pelo parceiro, porém a sincronização dos lotes é feita em intervalos de 30 minutos, portanto o parceiro deve ficar ciente que haverá um delay de no máximo 30 minutos até que as informações dos produtos enviados no lote estejam attualizadas no App.

Importante

Recomenda-se que os lotes sejam sempre compostos de até 1000 produtos.

Fluxo de Pagamentos

Ao finalizar o pedido com o cartão de crédito, nossa plataforma não captura o valor da compra do cliente imediatamente. Como durante a separação dos produtos podem haver faltas de mercadoria a captura só é feita após o pedido passar pelo PDV. Por isso, nos objetos Payment em nossos endpoint você verá o valor transaction_amount (que corresponde ao valor inicial do pedido) e o valor amount (referente ao valor capturado do pedido, após descontar todos os produtos faltantes).

Diferença entre Product e OrderItem

Dentro da nossa plataforma um objeto Product é um produto de fato, que pode ser um produto do banco de dados do App Super ou um produto que já foi designado para a tabela de um parceiro específico. Quando um cliente monta seu carrinho, e escolher um Product em um determinado estabelecimento, cria-se um novo objeto referente ao produto dentro de um pedido, o objeto OrderItem.
