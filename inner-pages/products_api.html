API de Produtos

Pela API de pedidos você pode acessar detalhes de pedidos, deletar pedidos, cancelar pedidos e finalizar pedidos, abaixo estão listado os endpoints disponíveis.

  GET     Retorna uma lista com os detalhes de produtos de um determinado estabelecimento.

/market/{market_id}/product-sku/

    Neste endpoint é obrigatório o fornecimento do identificador único do estabelecimento (market_id) para o qual você quer consultar o produto.

  • Resposta ( status_code = HTTP_200_OK )

    A Tabela 10 mostra a referência aos parâmetros inclusos na resposta desse endpoint.

    Tabela 14

  POST     Cria um novo lote de importação.

/market/{market_id}/product/

  • Parâmetros de Entrada

    Neste endpoint é possível criar lotes de importação para disponibilizar os produtos do seu estabelecimento no App Super, recomenda-se que os lotes de importação sejam de 1000 em 1000 produtos. O produto deve ter um identificador único especificado através do campo SKU. Além disso, deve-se fornecer obrigatoriamente identificador único do estabelecimento (market_id) juntamente com o objeto da Tabela 15. (Veja aqui Exemplo de Requisição)

    Atenção com EANs de Entrada

    Em alguns estabelecimentos existe o conceito de EAN de entrada. Isto é, alguns produtos possuem um EAN principal que se converte em EANs derivados. Isso acontece geralmente com produtos pesados. Por exemplo, o mercado recebe um queijo inteiro e as fatias de queijo tem um EAN derivado. No lote de produto deve ser mandado o EAN derivado e não o de entrada. Caso contrário haverá problema na catalogação dos produtos.

    Objeto Parâmetro Descrição
    ProductListAdd - -
    list [ProductAdd] product_list Lista de SKU dos produtos para importar
    (tamanho de lote recomendado: 1000 produtos)
    ProducAdd - -
    string name Nome do produto
    boolean bulk Booleano que identifica se um produto é ou não à granel.
    string
    (decimal)     		amount_in_stock Quantidade em estoque
    string barcode Código de barras
    string
    (decimal)     		price Preço
    string
    (decimal)     		low_price Preço com desconto
    string physical_location Localização no estabelecimento. Este campo serve para estabelecimentos que tem o endereçamento dos produtos nas gôndolas, caso seu estabelecimento não tenha, mandar uma string vazia ("").
    string sku Código reduzido do produto (SKU)
    boolean group_sale_required Venda em combo obrigatória
    object parent_skus Lista de skus pais
