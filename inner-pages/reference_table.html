  • Status de Pagamentos

No fluxo de pagametos do App Super, quando o cliente finaliza o pedido, fazemos uma pré-autorização do valor da compra. O valor só é realmente capturado quando a compra foi coletada e todas as ocorrências foram registradas. Assim, ao passar pelo ponto de venda (PDV) o valor pré-autorizado é ajustado, levando em conta os cortes, e então este valor ajustado é capturado. A Tabela 23 mostra os status de pegamentos que nossa API contempla.

Valor Status (Hard Coded) Descrição (status_display)
0 TRANSACTION_SUCCESS Pagamento pré-autorizado
1 TRANSACTION_ERROR Erro ao processar pagamento
2 TRANSACTION_CAPTURE_SUCCESS Pagamento capturado com sucesso
3 TRANSACTION_CAPTURE_ERROR Erro ao executar captura
4 TRANSACTION_CANCELED Pagamento cancelado
5 TRANSACTION_STATUS_NOT_IDENTIFIED Status não identificado
