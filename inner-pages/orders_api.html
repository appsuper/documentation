API de Pedidos

Pela API de pedidos você pode acessar detalhes de pedidos, deletar pedidos, cancelar pedidos e finalizar pedidos, abaixo estão listado os endpoints disponíveis.

  GET     Retorna lista de pedidos de um mercado específico.

/market/{market_id}/order/

  • Parâmetros de Entrada

    Neste endpoint é obrigatório o fornecimento do identificador único do estabelecimento (market_id) para o qual você quer a lista de pedidos. Além disso, se necessário é possível fornecer como dado de entrada para esse endpoint um parâmetro querystring de paginação para o resultado, por padrão os resultados são mostrados com paginação de 100 itens. Os pedidos retornados nesse endpoint são pedidos que passaram ou estão no status "Coletado". Portanto, os status visíveis nesse endpoint são os seguintes (Verificar tabela de referência para detalhes): Coletado, Pedido recebido pelo entregador, Pronto para entrega, Pronto para retirada, Saiu para entrega, Pedido de entrega aceito pelo entregador, Entregue, Pagamento Finalizado.

  • Resposta ( status_code = HTTP_200_OK )

    A Tabela 3 mostra a referência aos parâmetros inclusos na resposta desse endpoint.

  POST     Cancela um pedido.

/market/{market_id}/order/order-number/{order_number}/cancel/

  • Parâmetros de Entrada

    Neste endpoint é obrigatório o fornecimento do identificador único do estabelecimento (market_id) para o qual você quer cancelar o pedido e o número do pedido (order_number) a ser cancelado.

  • Resposta ( status_code = HTTP_201_CREATED )

    Caso o cancelamento do pedido seja bem sucedido esses endpoint retorna o status HTTP_201_CREATED.
